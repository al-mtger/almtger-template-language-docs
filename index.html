Almtger Template Language syntax reference

<{ Teach your HTML to think }>

A small, readable syntax for embedding logic in HTML — statements live inside <{ }>, output lives inside {{ }}. Nothing else to learn before you start reading templates.

<{ }> Template
<h1>Hi, {{ user.name }}</h1>
<{ if cart.items.length() > 0 }>
<p>You have {{ cart.items.length() }} items waiting</p>
<{ end if }>
→ Output

Hi, Hero

You have 2 items waiting

01 — Overview

Introduction

This is a lightweight template language for generating HTML — the same idea behind Django templates, Jinja, or Liquid, with its own compact syntax.

Every template is plain HTML with two kinds of markers dropped in. Statement tags (<{ … }>) handle logic: echoing values, assigning variables, branching, and looping. Echo brackets ({{ … }}) are a shorthand just for printing a value — the one thing you'll write the most. Everything outside these markers is passed through untouched.

02 — At a glance

Simple Example

A short menu template: loop over a list, echo a couple of fields for each item.

Given items is a list of { name, price } objects.

<{ }> Template
<ul>
<{ for item in items }>
<li>{{ item.name }} — ${{ item.price }}</li>
<{ end for }>
</ul>
→ Output
  • Espresso — $3.5
  • Latte — $4.25
  • Cold Brew — $4
03 — Reference

Syntax

Everything the language supports, grouped by what it does: printing values, assigning variables, branching, and looping.

Echo Statement

<{ echo expression }> evaluates expression and prints the result in place.

<{ }> Template
<{ echo user.name }>
→ Output

Hero

Procedures & Calculations

An expression can combine values with arithmetic, comparison, and logical operators.

Given price = 5.5, quantity = 5, age = 21, hasLicense = true.

<{ }> Template
<p>Total: ${{ price * quantity }}</p>
<{ if age >= 18 and hasLicense }>
<p>Eligible to drive</p>
<{ end if }>
→ Output

Total: $27.5

Eligible to drive

Fallback (OR)

The or operator works like Python's short-circuit evaluation: if the left-hand side is null (or missing), the expression resolves to the right-hand side. This is the idiomatic way to provide default values.

Given nickname may be null, username = "hero".

<{ }> Template
<p>Hello, {{ nickname or "guest" }}</p>
<p>Display: {{ username or "Anonymous" }}</p>
→ Output
nickname is null

Hello, guest

username = "hero" (not null)

Display: hero

How it works — x or "fallback" returns x if x has a value; otherwise it returns "fallback". This is especially useful for optional fields, labels, and placeholder text.

Chaining

Use ?. between links in a path to access nested values safely. If a link in the chain is missing, it returns null instead of raising an error. You can combine this with or to provide a fallback value.

<{ }> Template
<p>City: {{ user?.address?.city }}</p>
<p>City: {{ user?.address?.city or "Not provided" }}</p>
→ Output
user.address exists → city = "Cairo"

City: Cairo

City: Cairo

user.address is missing → resolves to null

City: 

City: Not provided

Why it matters — without ?., reading a property off a missing value stops the whole expression. Chaining lets you write one line instead of nesting an if around every optional field. Pair it with or for clean fallback values.

Simpler Version

{{ expression }} is shorthand for <{ echo expression }>. Same result, less to type — this is the form you'll reach for almost everywhere.

<{ echo total }> is the same as {{ total }}

Functions

Expressions can call a small set of built-in functions on a value, using dot syntax — value.function().

Length

variable.length() returns the number of items in a list, or the number of characters in a string.

Given items has 3 entries, username = "hero".

<{ }> Template
<p>You have {{ items.length() }} items in your cart</p>
<p>Username "{{ username }}" is {{ username.length() }} characters long</p>
→ Output

You have 3 items in your cart

Username "hero" is 4 characters long

Assign Statement

<{ let variable = expression }> evaluates the expression and stores it in a named variable. You can then use variable anywhere later in the template — inside echoes, conditions, or further assignments.

Given price = 49.99, quantity = 3, status may be null.

<{ }> Template
<{ let subtotal = price * quantity }>
<{ let shipping = 5 }>
<{ let total = subtotal + shipping }>
<{ let label = status or "Active" }>

<p>Subtotal: ${{ subtotal }}</p>
<p>Shipping: ${{ shipping }}</p>
<p><strong>Total: ${{ total }}</strong></p>
<p>Status: {{ label }}</p>
→ Output

Subtotal: $149.97

Shipping: $5

Total: $154.97

Status: Active

Tip — Variables are scoped to the template. Assign once and reuse anywhere below the assignment. This keeps your templates DRY and makes complex calculations readable.

If Condition

Branch with <{ if expression }>, optional <{ else if expression }> branches, an optional <{ else }>, and a closing <{ end if }>.

Given stock = 3.

<{ }> Template
<{ if stock == 0 }>
<span>Out of stock</span>
<{ else if stock < 5 }>
<span>Only {{ stock }} left</span>
<{ else }>
<span>In stock</span>
<{ end if }>
→ Output
Only 3 left

For Condition

Loop over a list with <{ for item in items }> … <{ end for }>. Everything between the two tags repeats once per item, with item scoped to that iteration.

Given post.tags = ["design", "frontend", "css"].

<{ }> Template
<ul>
<{ for tag in post.tags }>
<li>#{{ tag }}</li>
<{ end for }>
</ul>
→ Output
  • #design
  • #frontend
  • #css

Loop Variable

Add a name before the item — <{ for loop, item in items }> — to get a loop object alongside it. loop.index holds the current position.

Given instructions is a list of 3 strings.

<{ }> Template
<ol>
<{ for loop, step in instructions }>
<li>Step {{ loop.index }}: {{ step }}</li>
<{ end for }>
</ol>
→ Output
  1. Step 1: Preheat the oven
  2. Step 2: Mix the ingredients
  3. Step 3: Bake for 20 minutes
Note — loop.index starts at 1, so it prints ready to read — no need to add 1 to it yourself.
04 — Putting it together

Examples

Three complete templates that combine echoing, assigning, branching, chaining, and loops the way you'd actually use them.

Product grid with stock state

Given products is a list of { name, price, stock }.

<{ }> Template
<div class="grid">
<{ for product in products }>
<div class="card">
<h3>{{ product.name }}</h3>
<p>${{ product.price }}</p>
<{ if product.stock > 0 }>
<button>Add to cart</button>
<{ else }>
<button disabled>Sold out</button>
<{ end if }>
</div>
<{ end for }>
</div>
→ Output

Espresso

$3.5
Add to cart

Cold Brew

$4
Sold out

Greeting card with a guest fallback

Given user may or may not be signed in.

<{ }> Template
<div class="greeting">
<{ if user?.name }>
<h2>Welcome back, {{ user.name }}</h2>
<{ else }>
<h2>Welcome, guest</h2>
<{ end if }>
</div>
→ Output
Signed in

Welcome back, Hero

Signed out

Welcome, guest

Pagination with the current page highlighted

Given pages = [1,2,3,4,5], current = 3.

<{ }> Template
<nav>
<{ for loop, page in pages }>
<{ if loop.index == current }>
<span class="active">{{ page }}</span>
<{ else }>
<a href="?p={{ loop.index }}">{{ page }}</a>
<{ end if }>
<{ end for }>
</nav>
→ Output
1 2 3 4 5
